# ⚙️ Workflow: Skill Forge Pipeline

## 🎯 Objective

Orchestrate the end-to-end creation of a new gemini-kit skill. This pipeline enforces a strict transition from an interactive requirement gathering phase to an automated file scaffolding phase.

## 🔄 Phase 1: Interactive Requirement Gathering

**Actor:** `skill-analyst` (using `agents/skill-analyst.md`)

**Execution Loop:**

1. Feed the user's input and current chat history to the `skill-analyst`.
2. Evaluate the `skill-analyst`'s output:
   - **IF** the output ends with `>> CLI_PROMPT: WAITING_FOR_USER`: The system MUST pause the LLM stream, display the output to the user, prompt them for a reply in the terminal, and feed the new reply back into the `skill-analyst` context.
   - **IF** the output ends with `>> STATUS: REQUIREMENTS_GATHERED`: The MVS is complete. Capture the Requirement Spec generated by the analyst and immediately proceed to Phase 2.

## 🏗️ Phase 2: Architecture Design & Scaffolding

**Actor:** `planner` (using `agents/planner.md`)

**Execution Context:**
The `planner` is provided with the final Requirement Spec from Phase 1.

**Action:**

1. The `planner` acts as the Architect. It must design the content for the Master Bootstrapper (`skills/<skill-name>/SKILL.md`) and any necessary sub-documents (`references/*.md`) based on the spec.
2. The `planner` MUST NOT output the markdown files loosely. It MUST generate exactly ONE executable bash script block using `mkdir` and `cat << 'EOF'` to physically scaffold the files.

**Formatting Enforcement:**
The `planner` MUST format the generation exactly like this:

```bash

# 1. Create directories

mkdir -p skills/<skill-name>/references

# 2. Scaffold Master Bootstrapper

## cat << 'EOF' > skills/<skill-name>/SKILL.md
---
name: <skill-name>
description: <short-description>
version: 1.0.0
---

# 🛠️ Skill: <Skill Title>

... (content mapped from spec)
EOF
```

3. The `planner` MUST end its response exactly with the following flag to pause the system and request user permission before running the bash script:
   `>> CLI_PROMPT: EXECUTE_CONFIRMATION`
